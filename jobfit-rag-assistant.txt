Phase 2 — Build Plan: 5 Days → JobFit Assistant v1.0
Day 1 — llm.py (LLM Generation) ✅ COMPLETE (2026-06-22)
Concept Applied	Module
Temperature, top-k, top-p, decoding	llm.py
- Pick a local LLM: Microsoft Phi-3-mini (3.8B, best quality for laptop) or SmolLM2 (1.7B, faster)
  → DECISION: SmolLM2-1.7B-Instruct (lighter, runs on CPU, no quantization needed)
- Implement generate_text(prompt, temperature, top_k, top_p) with HuggingFace AutoModelForCausalLM
  → Done in src/jobfit/llm.py — lazy loading, chat template, AutoModelForCausalLM
- Use case: "Given this JD gap, suggest how to improve resume bullet XYZ"
  → Done in analyzer.py lines 61-72 — finds low-matching JD sentences, feeds to generate_text()
- Temperature experiment: Ran 3 temperatures (0.2, 0.7, 1.2) on Colab GPU → outputs in experiments/temperature_comparison.md
- Colab notebook: notebook/test_colab.ipynb — Cells: load model, temperature test, full pipeline
- Key insight: 0.7 is best default — balances coherence with useful variation
Day 2 — prompts.py (Structured Extraction)
Concept Applied	Module
Zero-shot, few-shot, JSON output	prompts.py
- Template library: extract_skills_prompt, summarize_role_prompt, gap_analysis_prompt
- extract_structured_jd(jd_text) → dict (role, skills, experience, tools)
- Few-shot examples for reliable JSON
- Integrate into analyzer.py — add "structured JD" section to report
Day 3 — function_caller.py (Tool Use / Agent Loop)
Concept Applied	Module
Tool calling, JSON schemas	function_caller.py
- Define 3 tool schemas:
- calculate_match_score(resume_path, jd_path)
- summarize_jd(jd_text)
- extract_skills(text)
- Build agent_loop(user_query) → LLM decides which tool → executes → returns result
- This replaces the hardcoded analyzer.py flow with an LLM-driven flow
Day 4 — validator.py (Hallucination Control)
Concept Applied	Module
System prompts, output constraints, grounded responses	validator.py
- validate_against_resume(llm_claim, resume_text) → bool — ensure LLM never claims skills not in resume
- validate_json(output, expected_schema) → bool — ensure structured output matches schema
- System prompt templates with strict constraints
- Add to analyzer.py as a post-processing step
Day 5 — vector_store.py (ChromaDB + RAG)
Concept Applied	Module
Embeddings, vector search, semantic retrieval	vector_store.py
- Store resume bullets + project descriptions as ChromaDB collection
- retrieve(query, k=5) → list[documents] — given a JD requirement, return top-K relevant bullets
- Replace the current similarity.py loop with a retrieval step
- Full RAG pipeline: Retrieve → LLM → Generate insight
Final Integration — analyzer.py v2
The new flow:
JD text
  → extract_structured_jd (prompts.py + llm.py)
  → retrieve relevant resume bullets (vector_store.py)
  → calculate_match_score (function_caller.py)
  → generate gap analysis + suggestions (llm.py + prompts.py)
  → validate against resume (validator.py)
  → print report
